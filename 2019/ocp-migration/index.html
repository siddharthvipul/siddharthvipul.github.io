Migrating jobs to new OCP4 cluster

Written by SiddharthVipul with ♥ on June 21, 2019 in

Action

If you are currently using apps.ci.centos.org (OCP 3.6), please contact us so that we can create a namespace for you in the new OCP 4.4 cluster for you to start migrating your jobs.

If your own a namespace on apps.ci.centos.org

There shouldn’t be a lot of changes for you if you have a project in our older openshift deployment (OCP 3.6)

pre-requisites

Before migrating, please gather all the requirements:

  • Do you need any special privileges in the namespace
  • All the jenkins plugins you need to run your job
  • do you need ability to checkout duffy nodes? (if yes, do you already have a duffy API key (just yes/no with the project name))

Things we need to create you a namespace (open a ticket with)

namespace name:
namespace admins:
 - user1
 - user2

You will have all the namespace level access and you own/manage pods/services inside it.

If your project is on ci.centos.org

Since you are using central jenkins master, your project would see some changes. You will have your own jenkins deployment to run all your jobs instead of having a shared jenkins instance where you don’t have admin access. With admin access, you can install all the plugins you need in jenkins, add credentials as it suits you and other abilities. This also means there is no direct maintenance from CentOS CI Infra admins as openshift takes care of things dying or crashing (but we will help you get started and are always in reach if you notice any hiccups that’s not auto healed). You can have the same workflow or use this opportunity to change your freestyle jobs to pipelines but that’s totally on you.
